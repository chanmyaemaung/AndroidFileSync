# Security Policy

## Supported Versions

We currently provide security updates for the following versions of AndroidFileSync:

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a high priority for AndroidFileSync. If you discover a security vulnerability within this project, please **do not** disclose it publicly by creating a public GitHub issue. 

Instead, please report it privately:

1. **Email:** Send your report to santoshmorya400@gmail.com.
2. **GitHub Security Advisories:** If enabled on this repository, navigate to the `Security` tab and click `Report a vulnerability`.

### What to include in your report
* A detailed description of the vulnerability.
* Steps to reproduce the issue.
* Any potential impact or risk you have identified.

### Response Time
We aim to acknowledge receipt of your vulnerability report within 48 hours and will provide a status update as we investigate and develop a patch. We ask that you give us a reasonable amount of time to fix the issue before making any public disclosure.

Thank you for helping keep AndroidFileSync secure!
